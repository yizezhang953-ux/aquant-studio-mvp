# 量化模型自由编辑平台 PRD

版本：v0.1  
阶段：第一阶段 - 需求定义  
日期：2026-06-20  
文档目标：明确产品边界、目标用户、核心场景、MVP 范围与后续交付依据。

## 1. 产品概述

本产品是一个面向个人交易者、量化研究者和小型投资团队的量化策略设计平台。用户可以根据自己的交易风格，自由选择交易频率、市场标的、技术指标、因子条件、买卖规则、仓位规则和风控规则，生成可回测、可比较、可迭代的量化模型。

第一阶段的产品边界以“策略设计 + 历史回测 + 结果分析”为核心，不直接进入真实交易执行。实盘交易、高频 Tick 级撮合、托管资金、投资建议服务等能力放入后续阶段。

## 2. 产品定位

### 2.1 产品愿景

让没有完整编程能力的交易者，也能像搭积木一样设计自己的量化模型；让具备编程能力的用户，可以在可视化编辑器和代码扩展之间自由切换。

### 2.2 产品形态

第一版建议做成 Web 应用：

- 浏览器访问，无需本地安装复杂环境。
- 后端负责数据、回测、任务调度与策略存储。
- 前端提供策略编辑器、回测配置、图表分析和模型管理。

### 2.3 核心价值

- 降低量化策略设计门槛。
- 支持不同交易风格：短线、波段、中长期、趋势、均值回归、多因子。
- 让策略在实盘前经过历史数据验证。
- 将策略、参数、回测结果和版本沉淀为可复用资产。

## 3. 目标用户

### 3.1 核心用户

| 用户类型 | 特征 | 核心需求 |
|---|---|---|
| 个人交易者 | 有交易经验，但不一定会写代码 | 用可视化方式设计策略并回测 |
| 量化初学者 | 懂基础指标，正在学习量化 | 使用模板理解策略逻辑 |
| 半专业研究者 | 会 Python 或有数据分析能力 | 快速验证模型和参数 |
| 小型投资团队 | 需要多人管理策略 | 管理策略版本、回测报告和风控规则 |

### 3.2 第一版优先用户

MVP 优先服务“个人交易者”和“量化初学者”。原因是这两类用户最需要低门槛策略编辑器，也最适合用模板和可视化回测形成早期产品反馈。

## 4. 使用场景

### 场景 1：用户创建一个均线策略

用户选择一个股票或加密货币标的，设置日线频率，添加规则：

- 当 MA5 上穿 MA20 时买入。
- 当 MA5 下穿 MA20 时卖出。
- 单次买入使用账户资金的 30%。
- 最大回撤超过 10% 时停止交易。

系统运行历史回测，输出收益曲线、最大回撤、胜率、交易明细和风险指标。

### 场景 2：用户选择模板并修改参数

用户从模板库选择“趋势跟随策略”，修改交易频率、指标周期、止损比例和持仓比例，然后保存为自己的策略版本。

### 场景 3：用户比较多个模型

用户选择 3 个策略版本，比较不同参数下的年化收益、最大回撤、夏普比率、交易次数和手续费影响。

## 5. 产品边界

### 5.1 MVP 范围内

第一版必须完成：

- 用户可以创建、编辑、保存、复制和删除策略。
- 用户可以选择交易市场、标的、时间范围和数据频率。
- 用户可以通过可视化规则编辑器组合指标条件。
- 用户可以设置买入、卖出、仓位和基础风控规则。
- 用户可以运行历史回测。
- 用户可以查看回测报告，包括收益曲线、回撤曲线、关键指标和交易记录。
- 系统提供基础策略模板。

### 5.2 MVP 暂不包含

第一版不做：

- 真实资金实盘交易。
- 高频 Tick 级交易。
- 券商账户托管。
- 投资组合自动推荐。
- 社区跟单。
- 策略售卖市场。
- 复杂机器学习训练平台。
- 多人协作权限体系。

### 5.3 后续可扩展

- 模拟盘。
- 实盘交易接口。
- 参数优化。
- AI 策略生成助手。
- 多资产组合回测。
- 策略市场。
- 团队协作与审计。

## 6. MVP 功能需求

## 6.1 用户账户

### 功能说明

用户可以注册、登录和管理自己的策略。

### MVP 要求

- 支持邮箱注册和登录。
- 支持找回密码。
- 每个用户只能查看自己的策略和回测记录。

### 验收标准

- 用户可以成功注册并登录。
- 未登录用户不能访问策略编辑和回测页面。
- 用户 A 不能查看用户 B 的策略。

## 6.2 策略管理

### 功能说明

用户可以创建、保存、复制、删除和查看策略版本。

### MVP 要求

- 策略名称。
- 策略描述。
- 交易市场。
- 数据频率。
- 策略规则。
- 仓位规则。
- 风控规则。
- 创建时间与更新时间。

### 验收标准

- 用户可以创建一个空白策略。
- 用户可以基于模板创建策略。
- 用户修改策略后可以保存新版本。
- 用户可以复制已有策略进行二次编辑。

## 6.3 数据选择

### 功能说明

用户可以选择回测所需行情数据。

### MVP 要求

建议第一版只选择一个主市场，避免范围过大。推荐二选一：

- 方案 A：中国 A 股日线与分钟线。
- 方案 B：加密货币现货日线与分钟线。

MVP 支持：

- 单标的回测。
- 日线。
- 1 小时线。
- 15 分钟线。
- 自定义回测时间范围。

暂不支持：

- Tick 数据。
- Level 2 订单簿。
- 多市场跨品种套利。

### 验收标准

- 用户可以选择标的。
- 用户可以选择频率。
- 用户可以选择回测开始日期和结束日期。
- 系统能校验数据是否存在。

## 6.4 策略编辑器

### 功能说明

策略编辑器是本产品的核心。第一版使用“条件组合式编辑器”，让用户用下拉框、输入框和条件组搭建策略。

### MVP 指标库

第一版内置：

- MA 简单移动平均线。
- EMA 指数移动平均线。
- MACD。
- RSI。
- Bollinger Bands 布林带。
- 成交量。
- 价格涨跌幅。
- 波动率。

### MVP 条件类型

支持：

- 大于。
- 小于。
- 等于。
- 上穿。
- 下穿。
- 连续 N 根 K 线满足。
- 任意条件满足。
- 全部条件满足。

### 买入规则

用户可以设置：

- 买入条件组。
- 买入价格类型：收盘价、下一根开盘价。
- 买入仓位比例。

### 卖出规则

用户可以设置：

- 卖出条件组。
- 止盈比例。
- 止损比例。
- 持有 N 根 K 线后卖出。

### 验收标准

- 用户可以创建至少一个买入条件和一个卖出条件。
- 用户可以组合多个条件。
- 系统可以将可视化规则保存为结构化 JSON。
- 回测引擎可以读取该 JSON 并执行策略。

## 6.5 仓位与风控

### 功能说明

用户可以设置基础资金管理规则。

### MVP 要求

- 初始资金。
- 单次交易仓位比例。
- 最大持仓比例。
- 手续费率。
- 滑点比例。
- 最大回撤停止交易。
- 单笔止损。
- 单笔止盈。

### 验收标准

- 回测结果会扣除手续费。
- 回测结果会考虑滑点。
- 当策略触发止损、止盈或最大回撤规则时，系统会执行对应动作。

## 6.6 回测引擎

### 功能说明

系统根据历史数据、策略规则和交易成本模拟交易。

### MVP 要求

- 单标的回测。
- 支持多种频率。
- 支持手续费。
- 支持滑点。
- 支持基础仓位管理。
- 输出交易记录。
- 输出每日或每根 K 线的权益曲线。

### 核心指标

必须输出：

- 总收益率。
- 年化收益率。
- 最大回撤。
- 胜率。
- 盈亏比。
- 夏普比率。
- 交易次数。
- 平均持仓周期。
- 手续费总额。

### 验收标准

- 一个策略可以成功完成回测。
- 回测完成后生成稳定的指标结果。
- 回测失败时能显示明确错误原因。

## 6.7 回测报告

### 功能说明

用户通过图表理解策略表现。

### MVP 要求

- 收益曲线。
- 回撤曲线。
- 买卖点标记。
- 交易明细表。
- 核心指标卡片。
- 参数与规则摘要。

### 验收标准

- 用户可以打开历史回测报告。
- 用户可以看到策略规则、回测区间和主要指标。
- 用户可以查看每一笔交易的买入时间、卖出时间、价格、收益率和手续费。

## 6.8 策略模板库

### 功能说明

为初学者提供可直接使用和修改的策略模板。

### MVP 模板

第一版提供：

- 双均线趋势策略。
- RSI 超买超卖策略。
- MACD 趋势策略。
- 布林带均值回归策略。
- 成交量突破策略。

### 验收标准

- 用户可以从模板创建策略。
- 模板创建后可以自由修改。
- 每个模板包含默认参数和简短说明。

## 7. 非功能需求

## 7.1 性能

- 单标的、5 年日线回测应在 5 秒内完成。
- 单标的、1 年 15 分钟线回测应在 30 秒内完成。
- 回测任务超过 10 秒时应显示任务进度或运行状态。

## 7.2 可用性

- 策略编辑器不能要求用户先写代码。
- 所有参数需要有默认值。
- 用户保存策略前，系统应提示明显的规则错误。
- 回测失败时，错误信息需要能帮助用户修改策略。

## 7.3 可靠性

- 策略、回测记录和交易明细需要持久化存储。
- 回测任务失败不能影响用户已有策略。
- 同一策略、同一数据、同一参数的回测结果应可复现。

## 7.4 安全

- 用户数据隔离。
- 密码加密存储。
- API 接口鉴权。
- 防止用户提交任意危险代码。
- 第一版不允许用户直接执行自定义 Python 代码。

## 7.5 合规与风险提示

产品需要明确展示：

- 回测结果不代表未来收益。
- 平台不构成投资建议。
- 用户需要自行承担交易风险。
- 实盘功能上线前需要单独进行合规评估。

## 8. 信息架构

MVP 页面建议：

| 页面 | 内容 |
|---|---|
| 登录/注册 | 用户认证 |
| 策略列表 | 查看、创建、复制、删除策略 |
| 策略编辑器 | 数据选择、指标条件、买卖规则、仓位风控 |
| 回测配置 | 标的、频率、时间范围、初始资金、手续费、滑点 |
| 回测报告 | 收益曲线、回撤曲线、指标、交易明细 |
| 模板库 | 从内置模板创建策略 |

## 10. MVP 用户流程

1. 用户注册并登录。
2. 用户进入策略列表。
3. 用户选择“新建策略”或“从模板创建”。
4. 用户选择市场、标的和频率。
5. 用户配置买入规则、卖出规则、仓位规则和风控规则。
6. 用户保存策略。
7. 用户进入回测配置页，选择时间范围和交易成本。
8. 用户点击运行回测。
9. 系统生成回测报告。
10. 用户复制策略版本并继续优化。

## 11. 成功指标

### 产品指标

- 用户首次创建策略成功率 >= 80%。
- 用户首次完成回测成功率 >= 70%。
- 模板策略使用率 >= 50%。
- 策略保存后再次编辑率 >= 40%。

### 技术指标

- 日线回测成功率 >= 99%。
- 常规回测接口错误率 < 1%。
- 回测结果可复现。
- 核心页面首屏加载时间 < 3 秒。

## 13. 关键决策点

项目启动前需要确认：

1. 第一版优先支持 A 股还是加密货币。
2. 是否允许用户写代码，还是第一版只支持可视化编辑。
3. 是否需要移动端适配，还是先只做桌面 Web。
4. 数据源预算和可用性。
5. 是否从第一版开始设计未来实盘接口的扩展结构。

## 14. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 数据质量不稳定 | 回测结果不可信 | 优先选择稳定数据源，记录数据版本 |
| 策略编辑器过于复杂 | 用户不会用 | MVP 只做条件组合式编辑器 |
| 回测结果过度理想化 | 用户误判策略 | 加入手续费、滑点和风险提示 |
| 范围过大 | 项目延期 | 第一版不做实盘和 Tick 高频 |
| 用户过拟合参数 | 策略失效 | 后续加入样本外测试和参数稳定性分析 |

## 15. 第一阶段结论

第一阶段建议将产品边界明确为：

“一个支持用户自由定义基础量化策略，并能对单标的历史行情进行回测和分析的 Web 应用。”

MVP 不追求完整金融交易闭环，而是先验证三个核心问题：

1. 用户是否能顺利搭建自己的策略。
2. 系统是否能准确执行策略并生成回测。
3. 回测结果是否能帮助用户继续迭代模型。

只要这三个问题被验证，后续再扩展模拟盘、实盘交易、高频数据和 AI 策略助手，产品路线会更稳。
