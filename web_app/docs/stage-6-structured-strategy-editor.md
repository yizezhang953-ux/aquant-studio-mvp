# 阶段 6 交付物：结构化策略编辑器

版本：V2.1 Stage 6  
日期：2026-06-21  
状态：已完成结构化表单编辑器，并保留 JSON 高级编辑模式。

## 1. 阶段目标

降低用户编辑量化策略的门槛。上一阶段只能直接编辑 JSON，本阶段把常见策略字段拆成可操作表单，让用户可以通过输入框和下拉框修改股票、周期、入场条件、出场条件、仓位、手续费、滑点和风控参数。

## 2. 已交付内容

- 结构化编辑模式：`form`
- JSON 高级编辑模式：`json`
- 表单与策略 JSON 双向转换
- 模板名称前端清洗，避免旧模板乱码影响主要界面
- 入场/出场规则编辑：
  - 逻辑：`all` / `any`
  - 条件：收盘价大于、大于等于、小于、小于等于阈值
- 标的与周期编辑：
  - 股票代码
  - 频率：`1d`、`60m`、`30m`、`15m`
  - 起止日期
- 资金与执行编辑：
  - 初始资金
  - 单笔资金比例
  - 最大仓位比例
  - 手续费率
  - 滑点率
- 风险控制编辑：
  - 止损比例
  - 止盈比例
  - 最大回撤比例
- 保存后继续调用后端策略持久化接口并生成版本记录。

## 3. 页面行为

用户复制模板后，前端会把策略 JSON 提取成表单字段。用户修改表单并点击“保存策略”时，前端会重新生成策略 JSON，再通过：

```text
PUT /api/v1/strategies/{strategy_id}
```

保存到后端。后端会生成新版本，版本历史可以继续载入到编辑器。

用户也可以切换到 JSON 模式，直接编辑完整策略 JSON。

## 4. 验收标准

- 前端 TypeScript 检查通过。
- Vite 构建通过。
- 结构检查通过。
- 用户可以在表单模式修改策略核心字段。
- 用户可以切换到 JSON 模式继续高级编辑。
- 表单保存后后端可生成新策略版本。
