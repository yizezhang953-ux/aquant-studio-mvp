# AQuant Studio 用户使用手册

## 1. 产品简介

AQuant Studio 是一个面向 A 股量化策略设计、行情数据管理、策略回测和结果分析的 Web 应用。用户可以注册账户，基于策略模板创建自己的策略，调整交易参数，运行回测，并查看资金曲线、交易记录、回测对比和行情数据质量。

当前版本适合学习、原型验证和早期策略研究，不用于真实下单。

## 2. 访问地址

本地开发环境：

```text
前端页面：http://localhost:5173/
后端健康检查：http://localhost:8000/health
后端 API 文档：http://localhost:8000/docs
```

如果以后部署到线上，前端和后端地址会变成部署平台提供的网址。

## 3. 第一次使用流程

### 3.1 打开页面

在浏览器打开：

```text
http://localhost:5173/
```

如果页面无法打开，通常说明前端服务没有启动。

### 3.2 注册或登录

第一次使用时，先注册账户：

1. 选择注册模式。
2. 输入邮箱。
3. 输入密码。
4. 输入显示名称。
5. 点击注册。

注册成功后，系统会自动登录。以后可以直接用邮箱和密码登录。

### 3.3 初始化数据

如果系统里没有模板、行情或回测示例，需要先初始化数据库。可通过后端接口或维护命令完成。

维护命令：

```bash
python -m app.tasks.database_maintenance init
```

初始化后，系统会写入策略模板、示例行情和示例回测数据。

## 4. 页面区域说明

登录后主页面分为几个区域：

### 4.1 左侧账户与策略区

这里用于：

- 查看当前登录用户。
- 退出登录。
- 选择策略模板。
- 复制模板成为自己的策略。
- 查看自己已经创建的策略列表。

常用操作：

1. 在模板下拉框选择一个策略模板。
2. 点击“复制模板”。
3. 左侧策略列表会出现新策略。
4. 点击策略后，中间编辑器会加载它。

### 4.2 中间策略编辑器

这里用于编辑量化策略。

支持两种编辑方式：

- 表单编辑：适合普通用户，直接修改股票代码、频率、日期、入场条件、出场条件、仓位和风控。
- JSON 编辑：适合高级用户，直接查看或修改策略 JSON。

常见字段：

- 股票代码：例如 `600519.SH`、`000001.SZ`
- 频率：例如 `1d`
- 开始日期 / 结束日期：回测区间
- 入场条件：例如收盘价大于某个阈值
- 出场条件：例如收盘价小于某个阈值
- 初始资金：策略模拟资金
- 手续费率 / 滑点率：交易成本假设
- 止损 / 止盈 / 最大回撤：风险控制参数

编辑后可以：

1. 点击“同步到 JSON 预览”。
2. 点击保存。
3. 系统会生成新的策略版本。

### 4.3 右侧行情数据区

这里用于查看和管理行情。

主要功能：

- 选择股票标的。
- 选择行情频率。
- 查看最新收盘价、K 线数量、起止日期。
- 查看收盘价走势图。
- 查看数据质量检查结果。
- 手动导入行情。
- CSV 文本导入。
- CSV 文件上传。
- 查看导入历史和失败详情。

如果质量检查显示缺失交易日或 OHLC 异常，说明行情数据可能不完整，需要补充或修正。

### 4.4 回测结果区

点击“运行回测”后，这里会显示：

- 总收益
- 最大回撤
- 胜率
- 交易次数
- 资金曲线
- 交易明细

回测结果会保存到历史记录中。

### 4.5 回测历史与对比区

这里可以：

- 查看过去运行过的回测。
- 勾选多条回测进行对比。
- 比较收益、回撤、胜率、交易次数。
- 查看不同回测之间的参数差异。

适合用于比较不同策略参数的效果。

## 5. 典型使用案例

### 案例 1：从模板创建并回测一个策略

1. 打开 `http://localhost:5173/`。
2. 注册或登录。
3. 在左侧选择一个模板，例如“双均线趋势策略”。
4. 点击“复制模板”。
5. 在策略列表中选择新策略。
6. 在中间表单中修改股票代码、日期和交易条件。
7. 点击保存。
8. 点击“运行回测”。
9. 查看右侧或下方的回测结果。
10. 在回测历史中保留记录，后续可对比。

### 案例 2：导入自己的行情数据

1. 登录系统。
2. 在行情数据区找到 CSV 导入。
3. 准备 CSV，至少包含：

```csv
trade_time,open,high,low,close,volume,amount
2024-02-01,10,11,9.8,10.5,1000,10500
2024-02-02,10.5,10.8,10.2,10.6,1200,12720
```

4. 填写股票代码、名称和频率。
5. 粘贴 CSV 内容或上传 CSV 文件。
6. 点击导入。
7. 查看导入历史和质量检查结果。

### 案例 3：比较不同参数的策略效果

1. 选择一个策略。
2. 修改入场阈值，保存并运行回测。
3. 再修改另一个参数，保存并运行回测。
4. 在回测历史中勾选多个结果。
5. 查看回测对比和参数差异。

## 6. 数据维护

### 6.1 查看数据库状态

```bash
python -m app.tasks.database_maintenance status
```

或访问：

```text
http://localhost:8000/api/v1/database/status
```

### 6.2 初始化数据库

```bash
python -m app.tasks.database_maintenance init
```

### 6.3 备份数据库

```bash
python -m app.tasks.database_maintenance backup
```

备份文件会生成在：

```text
runtime/backups
```

线上部署时，如果平台磁盘不是持久化的，应及时下载备份文件。

## 7. 常见问题

### 页面打不开

检查前端服务是否运行。正常访问地址是：

```text
http://localhost:5173/
```

### 页面能打开，但数据加载失败

检查后端服务是否运行：

```text
http://localhost:8000/health
```

如果后端正常，会返回：

```json
{"status":"ok","environment":"local"}
```

### 登录后看不到策略

可以先复制一个模板。如果模板为空，先初始化数据库。

### 回测没有交易

可能原因：

- 入场条件从未触发。
- 回测日期范围太短。
- 股票代码没有对应行情。
- 出场条件设置过严。

可以降低入场阈值或扩大回测时间范围。

### 行情质量检查有 warning

warning 不一定代表系统错误，通常表示：

- 某些交易日缺失。
- 成交量为 0。
- 行情区间不连续。

如果要做更严肃的回测，应先补齐行情。

## 8. 当前版本边界

当前版本已经支持策略设计、保存、版本管理、行情导入、回测和部署准备。

暂未完成或仍处于预留状态的能力：

- 真实 Tushare/AkShare 网络行情拉取。
- 自动定时同步行情。
- 自动恢复数据库备份。
- 真实交易下单。
- 多用户权限分级管理。
- 完整线上域名和 HTTPS 部署。

真实交易功能默认禁用，不建议在当前版本中用于实盘。
