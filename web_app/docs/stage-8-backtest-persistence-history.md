# 阶段 8 交付物：回测结果持久化与历史记录

版本：V2.1 Stage 8  
日期：2026-06-22  
状态：已完成登录用户回测结果入库，以及前端“我的回测记录”。

## 1. 阶段目标

让回测不再只是一次性运行结果。用户登录后运行回测，系统会把回测报告、指标、交易明细和权益曲线保存到数据库，用户可以在前端查看历史回测并重新打开报告。

## 2. 已交付内容

- `backtest_runs` 增加：
  - `owner_id`
  - `source_strategy_id`
- SQLite 自动补字段迁移。
- 回测运行时带登录 token 会自动持久化。
- 新增 API：
  - `GET /api/v1/backtests`
  - `GET /api/v1/backtests/mine/{backtest_id}`
- 前端新增“我的回测记录”列表。
- 点击历史记录可重新载入对应回测报告。
- 后端测试覆盖登录用户回测入库、列表查询、详情查询。

## 3. 使用流程

1. 用户登录前端。
2. 选择自己的策略。
3. 点击“运行回测”。
4. 后端完成回测并写入数据库。
5. 前端刷新“我的回测记录”。
6. 用户点击历史记录，右侧重新展示该次回测报告。

## 4. 数据边界

- 未登录用户仍可调用旧回测接口运行临时回测，但不会进入“我的回测记录”。
- 登录用户只能查询自己的回测记录。
- 回测仍然是模拟环境，不涉及实盘交易。

## 5. 验收标准

- 后端测试通过。
- 前端 TypeScript 检查通过。
- Vite 构建通过。
- 结构检查通过。
- 登录用户运行回测后，可以通过历史接口查到结果。
